# [!DNL Workfront] のメンテナンス更新

2023年に行われたメンテナンス更新は次のとおりです。

>[!NOTE]
>
>これらの更新には、その他の軽微なバグ修正や、あまり目立たないバグ修正も含まれます。ユーザーが送信した問題が修正されると、[!DNL Workfront] サポートからそのユーザーに通知が届きます。

2023年より前のメンテナンス更新については、[以前のメンテナンス更新](#previous-maintenance-updates)を参照してください。

## 2023年2月の更新

+++**2023年23月2日のメンテナンス更新**

**コメントのリダイレクト先へのリンク [!UICONTROL 詳細] ページ**

_更新_

ユーザーが Workfront でオブジェクト上のコメントへのリンクに従うと、更新ストリームが簡潔に読み込まれ、その後、ユーザーがオブジェクトの[!UICONTROL 詳細]領域にリダイレクトされます。これは、ユーザーがメールからリンクをクリックしたり、ブラウザーにリンクを貼り付けたりすると発生する可能性があります。

これは、現在、ドキュメントオブジェクトにのみ影響します。

**ユーザーは自分の通知設定を編集できません**

_ユーザー_

[!UICONTROL 作業者]ライセンスを持つユーザーが独自の通知設定を編集しようとすると、[!UICONTROL 編集]ウィンドウに「[!UICONTROL 通知]」オプションが表示されず、ユーザーは設定を編集できません。

+++

+++**2023年16月2日のメンテナンス更新**

**ボードでの複数のチーム割り当て**

_ボード_

複数のチームをボード上のタスクやイシューに割り当てたり、ボード自体に割り当てたりできるようになりました。

**カードのフォールオフ制限の増加**

_ボード_

カードのフォールオフ時間制限は、4 週間/ 30 日ではなく 8 週間/ 60 日に増加しました。

**スケジュールされた非アクティブ化でユーザーが非アクティブ化されない**

_ユーザー_

ユーザーの非アクティブ化がスケジュールされている場合、スケジュールされた日時が経過すると、ユーザーは非アクティブ化されません。

+++

+++**2023年2月9日のメンテナンス更新**

タスクリスト、問題リストおよびレポートに追加された&#x200B;**[!UICONTROL ストーリーポイント]フィールド**

_レポート_

[!UICONTROL ストーリーポイント]フィールドが、タスクまたは問題のリストおよびレポートに追加できるようになりました。このフィールドは、予定時間またはチーム割り当てに結び付けられない、編集可能なフリーフォームフィールドです。

+++

+++**2023年2月8日のメンテナンス更新**

**取り込み列のフィルターボタン**

_ボード_

ボード上の取り込み列には、列が空でフィルターが作成されていない場合に「**[!UICONTROL フィルターを追加]**」ボタンが含まれるようになりました。このボタンで設定領域が開き、タスクおよび問題を取り込み列に入れるためのフィルターを追加できます。

+++

+++**2023年2月2日のメンテナンス更新**

デフォルトで、**[!UICONTROL ボード]アイコンが[!UICONTROL メインメニュー]に表示される**

_ボード_

レイアウトテンプレートのないユーザーに対して、[!UICONTROL ボード]アイコンが[!UICONTROL メインメニュー]に表示されるようになりました。また、ボードは、作成された新しいレイアウトテンプレートのメインメニューにもデフォルトで含まれます。既存のレイアウトテンプレートには、変更はありません。

**メールテンプレートを保存できない**

_設定_

ユーザーがメールテンプレートを作成または編集しようとすると、「[!UICONTROL 保存]」ボタンが応答しなくなり、ユーザーはテンプレートを保存できません。

+++

## 2023 年 1 月の更新

+++**2023年1月30日のメンテナンス更新**

**一般的なタイムシート操作用に追加されたキーボードショートカット**

_タイムシート_

タイムシート内で一般的に実行される操作用に、次のキーボードショートカットが導入されました。

* 行を追加（Command + Option + +／Ctrl + Option + +）
* 行を削除（Command + Option + -／Ctrl + Option + -）
* 作業アイテムをピン留めまたはピン留めを解除（Option + P／Option + P）
* コメントを開く（Shift + F2／Shift + F2）
* コメントを保存（Command + Enter／Ctrl + Enter）
* 展開（Shift + Option + 上矢印／Shift + Alt + 上矢印）
* 折りたたむ（Shift + Option + 下矢印／Shift + Alt + 下矢印）

これらの操作を適用するには、操作が実行される領域をハイライト表示する必要があります。

**タイムシート、タイムシートプロファイルおよびタイムシート環境設定の新規情報アイコン**

_タイムシート_

>[!NOTE]
>
>この更新は、2022年11月3日にプレビュー環境に対してのみリリースされ、現在は実稼動環境で利用できます。

次の設定に対して、いくつかの情報アイコンを追加しました。

* タイムシートやタイムシートプロファイルを作成または編集する際に、「[!UICONTROL 時間を編集可能]」チェックボックスを有効にすると、管理者が[!UICONTROL 設定]の[!UICONTROL タイムシート環境設定]領域でこれらのアクションを制限することなく、承認者もタイムシートを送信、再オープンまたは編集できます。
* [!UICONTROL 設定]の[!UICONTROL タイムシートおよび時間環境設定]領域の「[!UICONTROL タイムシートの変更を所有者と管理者に制限します]」を無効にすると、タイムシートおよび時間への管理アクセス権を持つユーザー、時間の編集を許可されたタイムシート承認者、タイムシート所有者のマネージャーもタイムシートを編集できます。

これらの設定の機能は変更されず、設定の範囲を明確にするために情報アイコンが追加されるだけであることに注意してください。

+++

+++**2023年1月26日のメンテナンス更新**

**[!DNL Outlook]** からリクエストを送信中にエラーが発生する

_統合_

ユーザーが、[!DNL Outlook] メールから添付ファイルを付けてリクエストを送信しようとすると、1 つ以上の添付ファイルがアップロードされず、次のエラーが表示されます。

「[!UICONTROL 次のエラーが発生しました：ハンドル xxxx を持つファイルが存在しません。]」

これは、リクエストキューから割り当てを行った場合、またはリクエスト作成時に手動で新しいリクエストに対し割り当てを行った場合にのみ発生します。

**新規デスクトッププルーフビューアバージョン**

_プルーフ_

デスクトッププルーフビューアでのフリーズの問題を修正するために、新しいバージョンのデスクトッププルーフビューアをデプロイしました。デスクトッププルーフビューアを既にインストールしているユーザーは、このアップデートを自動的に取得します。

また、ユーザーは、最新バージョンを手動でダウンロードすることもできます。詳しくは、[デスクトッププルーフビューアのインストール](https://experienceleague.adobe.com/docs/workfront/using/review-and-approve-work/proofing/use-the-desktop-proofing-viewer/installing-desktop-proofing-viewer.html?lang=ja)を参照してください。

* 以前のバージョン：2.1.19
* 新しいバージョン：2.1.20

**ユーザーが自身のユーザー設定を編集できない**

_ユーザー_

「ワーク」、「レビュー」または「リクエスト」のライセンスを持つユーザーが自身のユーザー設定を編集しようとすると、空白のポップアップが開き、編集できません。ポップアップを終了するには、ユーザーはページを更新する必要があります。

+++

+++**2023年1月19日のメンテナンス更新**

**取り込み列フィルターを共有できるようになりました**

_ボード_

取り込み列機能がボードにリリースされた場合、取り込み列を設定するためのフィルターは、それらのフィルターの作成者のみに表示されます。作成者は、他のユーザーやチームとフィルターを共有できるようになりました。

**[!UICONTROL その他]メニューで使用できるピン留め機能**

_ナビゲーション_

プレビュー環境の[!UICONTROL その他]メニューで、ピン留めに関する次の機能が使用できます。

* ピン留めの名前の変更
* [!UICONTROL その他]メニュー内でのピン留めの並べ替え
* ピン留めを[!UICONTROL その他]メニューから移動させる（これを行うと、上部のナビゲーションバーの最後のピン留めが[!UICONTROL その他]メニューに移動します）

+++

+++**2023年1月18日のメンテナンス更新**

**ワイルドカードを含む式は、カスタムフィールドでは無効です**

_カスタムフォーム_

ユーザーがカスタムフィールドで \$$TODAY や $$NOW などのワイルドカードと修飾子（「 -30d」など）を併用している場合、バリデータはそのワイルドカードを有効なものとして受け付けません。修飾子のないワイルドカードは有効と見なされます。

**[!UICONTROL ワークロードバランサー]に、プロジェクト、タスク、問題に関連付けられていない時間が表示される**

_[!UICONTROL ワークロードバランサー]_

ユーザーが[!UICONTROL ワークロードバランサー]を表示すると、プロジェクト、タスク、または問題に関連付けられていない、または、[!UICONTROL 一般的]な時間として記録されていない、ユーザー用に記録された時間が表示されます。この時間は、4 週間または 6 週間のビューでのみ表示されます。

+++

+++2023年1月12日 **[!DNL Adobe Workfront Fusion]のメンテナンス更新（ホットフィックス）**

**[!DNL Workfront] モジュールの 404 エラー**

_Workfront Fusion_

シナリオを実行すると、[!DNL Workfront] モジュールが 404 エラーを返します。

この問題は、次のモジュールで報告されています。

* [!UICONTROL レコードを読み取る]

+++

+++**2023年1月12日のメンテナンス更新（ホットフィックス）**

**計算フィールド設定時の「[!UICONTROL Whoops]」エラー**

_カスタムフォーム_

ユーザーがカスタムフォームで計算フィールドを作成または編集する際、計算フィールドの式にカスタムフィールドを含めると、式は無効と見なされます。「[!UICONTROL 保存]」ボタンは無効になっており、ユーザーはカスタムフィールドから移動できません。さらに、フィールドの下にエラーメッセージが表示されます。

「[!UICONTROL エラーが発生しました。Workfront にお問い合わせください。問題の原因を確認して修正させていただきます。]」

式からカスタムフィールドを削除すると、ユーザーはフィールドを保存して、フィールドから移動できるようになります。

**アクセスレベルを設定できません**

_ユーザー_

ユーザーが別のユーザーのアクセスレベルを変更しようとすると、アクセスレベルがグレー表示され、ユーザーは変更できません。これは、変更しようとしているユーザーがシステム管理者である場合でも発生します。

+++

+++**2023年1月12日のメンテナンス更新**

**ドロップダウンフィールドで Ctrl + F または Cmd + F キーが期待どおりに動作しない**

_カスタムフォーム_

ユーザーがカスタムフォームに入力し、Ctrl + F または Cmd + F キーを使用してドロップダウンリストを検索する際に、その検索の次のインスタンスにジャンプしようとすると、ドロップダウンリストは検索の次のインスタンスにジャンプせず、リストの先頭に戻ります。この問題は、ドロップダウンが複数選択を許可するように設定されている場合に発生します。

**[!UICONTROL レポートを編集]画面が空**

_レポート_

ユーザーがレポートを表示中に、レポートを編集しようとすると、空の画面が表示され、レポートを編集できません。

**インデントされたタスクのインデントが維持されない**

_タスク_

ユーザーがタスクリストを表示中に、タスクをインデントすると、タスクが即座に元の（インデント解除された）状態に戻ります。

+++

+++**2023年1月5日のメンテナンス更新**

**[!UICONTROL その他]メニューで使用できるピン留め機能**

_ナビゲーション_

プレビュー環境のみで、ピン留めに関する次の機能が[!UICONTROL その他]メニューで使用できます。

* ピン留めの名前の変更
* [!UICONTROL その他]メニュー内でのピン留めの並べ替え
* ピン留めを[!UICONTROL その他]メニューから移動させる（これを行うと、上部のナビゲーションバーの最後のピン留めが[!UICONTROL その他]メニューに移動します）

**プロジェクトチームへのユーザー追加からのプロジェクトグループの制限の撤廃**

_チーム_

プロジェクトチームに追加される必要があるユーザーは、プロジェクトに関連付けられたグループに属する必要があるという制限を撤廃しました。現在は、属しているグループにかかわらず、任意のアクティブユーザーをプロジェクトチームに追加できます。

**タイムシート、タイムシートプロファイルおよびタイムシート環境設定の新規情報アイコン**

>[!NOTE]
>
>この更新は、以前、2022年11月3日にプレビュー環境に対してリリースされており、現在は実稼動環境で利用できます。

_Workfront_

次の設定に対していくつかの情報アイコンを追加しました。

* タイムシートやタイムシートプロファイルを作成または編集する際に、「時間を編集可能」チェックボックスを有効にすると、設定の管理者がタイムシート環境設定領域でこれらのアクションを制限することなく、承認者もタイムシートを送信、再オープンまたは編集できます。
* 設定のタイムシートおよび時間環境設定領域の「タイムシートの変更を所有者と管理者に制限します」を無効にすると、タイムシートおよび時間への管理アクセス権を持つユーザー、時間の編集を許可されたタイムシート承認者、タイムシート所有者のマネージャーもタイムシートを編集できます。

これらの設定の機能は変更されず、設定の範囲を明確にするために情報アイコンが追加されるだけであることに注意してください。
