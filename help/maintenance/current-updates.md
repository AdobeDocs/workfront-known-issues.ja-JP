# [!DNL Workfront] のメンテナンス更新

2023年に行われたメンテナンス更新は次のとおりです。

>[!NOTE]
>
>これらの更新には、その他の軽微なバグ修正や、あまり目立たないバグ修正も含まれます。ユーザーが送信した問題が修正されると、[!DNL Workfront] サポートからそのユーザーに通知が届きます。

2023年より前のメンテナンス更新については、[以前のメンテナンス更新](#previous-maintenance-updates)を参照してください。

## 2023年7月の更新

+++**（予定） 2023 年 7 月 27 日のメンテナンス更新**

**&quot;[!UICONTROL システム全体を有効にする]&quot;および&quot;[!UICONTROL システム全体を表示]」は異なる機能を表します。**

_フィルター_

ユーザーがフィルターを共有し、「[!UICONTROL システム全体で表示]」オプションを有効にすると、フィルターはシステム内のすべてのユーザーと共有されます。ただし、管理者が[!UICONTROL 設定]でこのフィルターを表示すると、このフィルターが「[!UICONTROL システム全体で表示]」列に「[!UICONTROL false]」と表示されます。このフィルターをシステムのデフォルトにするには、管理者は[!UICONTROL 設定]で「[!UICONTROL システム全体で有効にする]」オプションを有効にする必要があります。これは、表現が類似しているので、混乱を招く可能性があります。

+++

+++**2023年7月20日（PT）のメンテナンス更新**

この更新には、軽微なバグ修正や、あまり目立たないバグ修正のみが含まれています。ユーザーが送信した問題が修正されると、[!DNL Workfront] サポートからそのユーザーに通知が届きます。

+++**2023年7月13日（PT）のメンテナンス更新**

**タイムラインが再計算されない**

_プロジェクト／タスク／イシュー_

タイムライン計算をトリガーするイベントが発生しても、タイムラインは再計算されません。これは、変更時に発生する再計算と、スケジュールされた再計算に影響します。ワークロードバランサーの精度に影響を与える可能性があります。

**ロックされたプルーフの承認が、作業リスト内に引き続き表示される**

_プルーフ_

期限を過ぎ、ロックされたプルーフの承認が、期限が過ぎてもリストからドロップされず、承認者のホームワークリストに引き続き表示される。

**稼働率レポートが読み込まれない**

_レポート_

顧客が稼働率レポートを表示しようとすると、回転している読み込みインジケーターが表示されますが、レポートは読み込まれません。レポートから 500 エラーが返されましたが、レポートに失敗したことを示す表示はありません。

**ユーザーを編集ページが空白である**

<!--no article-->

_ユーザー_

ユーザーが別のユーザーを編集しようとすると、ユーザーを編集ページが空白になり、そのユーザーは他のユーザーを編集できません。

**[!UICONTROL 承認待ち]ステータスでタスクの並べ替え時にエラーメッセージが表示されない**

_タスク_

ユーザーがタスクリスト内のタスクを並べ替えようとした際に、そのタスクが [!UICONTROL 承認待ち] ステータスにある場合、そのタスクはタスクリスト内で移動したように表示されます。更新すると、ユーザーには項目が移動していないことがわかります。項目は[!UICONTROL 承認待ち]ステータスにあるので移動できませんが、ユーザーに項目が移動できないことを示すメッセージが表示されないので、混乱が生じる可能性があります。

**依存タスクの下で先行タスクの移動時にエラーメッセージが表示されない**

_タスク_

ユーザーがタスクリスト内のタスクを並べ替えようとした際に、そのタスクが [!UICONTROL 承認待ち] ステータスにある場合、そのタスクはタスクリスト内で移動したように表示されます。更新すると、ユーザーには項目が移動していないことがわかります。先行タスクを、それよりさらに先行タスクの下に移動することはできません。これによって項目をできない場合に、ユーザーに項目が移動できないことを示すメッセージが表示されないので、混乱が生じる可能性があります。

## 2023年4月の更新

+++**2023年4月27日（PT）のメンテナンス更新**

**[!UICONTROL プルーフビューア]** でプルーフを切り替えることができない

+++**2023年4月20日（PT）のメンテナンス更新**

**カスタムドロップダウンフィールドのイシュー**

_カスタムフォーム_

複数選択フィールドとして有効になるカスタムドロップダウンフィールドには、次のイシューが表示される場合があります。

* フォームが編集モードでない場合、「+ [!UICONTROL 追加]」ボタンが表示されない。
* 値のないフィールドに、「-- [!UICONTROL ラベルなし] --」オプションが表示される。

**プルーフにコメントを作成する際にポリラインツールを使用できない**

_プルーフ_

ユーザーがプルーフビューアでプルーフを表示している際に、ポリラインツールを使用してコメントを作成しようとすると、ツールでプルーフがマークアップされません。

**テキストオプションボックスに「textAnnotations」と表示される**

_プルーフ_

ユーザーがプルーフを表示している際にコメントの追加を開始し、テキストツールを開くと、ツールのオプションの横に「textAnnotation」という単語が表示されます。テキスト ツールは引き続き期待どおりに機能し、コメントが投稿されると「textAnnotation」は表示されなくなります。

**コメント機能のベータ版エクスペリエンスでの目標やイシューの更新から離れる際に画像を下書きとして保持する**

>[!NOTE]
>
>この機能は、2023年4月19日（PT）にプレビュー用にリリースされ、2023年4月20日（PT）に実稼動用にリリースされました。

_アップデート_

画像を添付したメッセージの作成中に更新ページから離れても、そのページに戻ると、メッセージと画像が保持されるようになりました。この更新前には、未送信のコメントは保存されていましたが、画像は削除されていました。これは、目標に関する「更新」セクションと、コメント機能のベータ版エクスペリエンスを有効にする際のイシューの「更新」セクションで利用できます。

**「更新」セクションのリアルタイムの更新および削除されたコメント**

>[!NOTE]
>
>この機能は、2023年4月19日（PT）にプレビュー用にリリースされ、2023年4月20日（PT）に実稼動用にリリースされました。

_アップデート_

他のユーザーがコメントや返信を追加したり、更新エリアからコメントを削除したりすると、新しいコメントやコメントが削除されたことをリアルタイムで遅延なく確認できるようになりました。これは、目標に関する「更新」セクションと、コメント機能のベータ版エクスペリエンスを有効にする際のイシューの「更新」セクションで利用できます。

**変更記録なしで、システムによってアクセスレベルが変更される**

_ユーザー_

ユーザーのアクセスレベルは、システムによって予期せず変更される場合があります。これが発生すると、表示されるアップデートはなく、変更は監査ログに表示されません。

_ドキュメント_

これは、現在、ドキュメントオブジェクトにのみ影響します。

## 2023年1月の更新

+++**2023年1月30日（PT）のメンテナンス更新**

**一般的なタイムシート操作用に追加されたキーボードショートカット**

_タイムシート_

タイムシート内で一般的に実行される操作用に、次のキーボードショートカットが導入されました。

* 行を追加（Command + Option + +／Ctrl + Option + +）
* 行を削除（Command + Option + -／Ctrl + Option + -）
* 作業アイテムをピン留めまたはピン留めを解除（Option + P／Option + P）
* コメントを開く（Shift + F2／Shift + F2）
* コメントを保存（Command + Enter／Ctrl + Enter）
* 展開（Shift + Option + 上矢印／Shift + Alt + 上矢印）
* 折りたたむ（Shift + Option + 下矢印／Shift + Alt + 下矢印）

これらの操作を適用するには、操作が実行される領域をハイライト表示する必要があります。

**タイムシート、タイムシートプロファイルおよびタイムシート環境設定の新規情報アイコン**

_タイムシート_

>[!NOTE]
>
>この更新は、2022年11月3日（PT）にプレビュー環境に対してのみリリースされ、現在は実稼動環境で利用できます。

次の設定に対して、いくつかの情報アイコンを追加しました。

* タイムシートやタイムシートプロファイルを作成または編集する際に、「[!UICONTROL 時間を編集可能]」チェックボックスを有効にすると、管理者が [!UICONTROL 設定] の [!UICONTROL タイムシート環境設定] 領域でこれらのアクションを制限することなく、承認者もタイムシートを送信、再オープンまたは編集できます。
* [!UICONTROL 設定] の [!UICONTROL タイムシートおよび時間環境設定] 領域の「[!UICONTROL タイムシートの変更を所有者と管理者に制限します]」を無効にすると、タイムシートおよび時間への管理アクセス権を持つユーザー、時間の編集を許可されたタイムシート承認者、タイムシート所有者のマネージャーもタイムシートを編集できます。

これらの設定の機能は変更されず、設定の範囲を明確にするために情報アイコンが追加されるだけであることに注意してください。
